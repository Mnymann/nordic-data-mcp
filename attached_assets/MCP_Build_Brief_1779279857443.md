# Nordic Data MCP Server — Build Brief

**Til:** Replit Agent i et nyt, selvstændigt Replit-projekt
**Fra:** Martin Nymann (Addon Nordic ApS)
**Dato:** Maj 2026

Dette dokument er det komplette brief til at bygge `nordic-data-mcp` — en Model Context Protocol-server der eksponerer Nordic Data API'et til AI-agenter (Claude, Cursor, ChatGPT, Copilot m.fl.).

---

## 1. Hvad du bygger

En **tynd MCP-adapter** der oversætter Nordic Data API's HTTP-endpoints til MCP tool-kald. Den indeholder **ingen forretningslogik og ingen data** — den videresender bare AI-agentens kald til den eksisterende backend.

```
[Claude / Cursor / ChatGPT / Copilot]
              ↓ (MCP-protokol over stdio eller HTTP)
        nordic-data-mcp
        (DETTE PROJEKT — nyt, selvstændigt)
              ↓ (HTTP + X-API-Key header)
        api.addonnordic.dk
        (Nordic Data API — EKSISTERENDE, RØR IKKE)
```

**Vigtig afgrænsning:** Du bygger IKKE backend-API'et. Backend findes allerede og er live på `https://api.addonnordic.dk`. Du bygger kun MCP-wrappen.

## 4. Backend API — hvad du integrerer med

### Base URL
- Production: `https://api.addonnordic.dk`
- Lokal udvikling (hvis Martin giver adgang): N/A — brug altid prod

### Authentication
Alle `/api/*` kald kræver header:
```
X-API-Key: <NORDIC_API_KEY>
```

### Coverage
14 EU-lande er live: `DK, NO, SE, FI, NL, BE, IE, UK, FR, DE, CZ, PL, LV, EE`

### Endpoints v1 eksponerer

| MCP-tool | HTTP-metode + path | Note |
|---|---|---|
| `lookup_company` | `GET /api/company/{country}/{id}` | 14 lande, ID-format varierer per land |
| `validate_vat` | `GET /api/vat/validate/{country}/{vat}` | VIES for EU; HMRC for GB (sandbox-default) |
| `screen_sanctions` | `POST /api/sanctions/screen` | Bulk-screen op til 1000 navne |
| `kyb_full` | `GET /api/kyc/full/{country}/{id}` | Master KYB report — identity + persons + financials + LEI + VAT + sanctions + adverse media + risk score |
| `autocomplete_address` | `GET /api/address/{country}/autocomplete?q={query}` | DAWA (DK), Kartverket (NO), Nominatim (others), BAN (FR), PDOK (NL) |
| `lookup_lei` | `GET /api/lei/{lei}` + `GET /api/lei/lookup/{country}/{id}` | GLEIF — forward + reverse |
| `company_enriched` | `GET /api/company/{country}/{id}/enriched` | Wikidata + adresse + industry stats |

### Country/ID-format reference

| Land | ID-type | Regex / format |
|---|---|---|
| DK | CVR | 8 cifre |
| NO | Organisasjonsnummer | 9 cifre |
| SE | Organisationsnummer | 10 cifre (med eller uden bindestreg) |
| FI | Y-tunnus | 7 cifre + bindestreg + 1 check-ciffer (`0101120-3`) |
| NL | KVK | 8 cifre |
| BE | Enterprise number | `BE` + 10 cifre (eller bare 10 cifre) |
| IE | CRO number | 1-7 cifre |
| UK | Companies House number | 8 tegn (cifre eller præfiks som `SC`, `NI`, `OC`) |
| FR | SIREN | 9 cifre |
| DE | LEI eller HRB | LEI = 20 alphanum; HRB = præfiks + cifre |
| CZ | IČO | 8 cifre |
| PL | NIP / REGON / KRS | NIP=10, REGON=9/14, KRS=10 |
| LV | Reģistrācijas nr. | 11 cifre |
| EE | Registrikood | 8 cifre |

### Response-konventioner
- Success: JSON-body, status 200
- Not found: status 404, `{"error":"not_found","country":"...","id":"..."}`
- Invalid input: status 400, `{"error":"invalid_id_format","expected":"..."}`
- Upstream nede: status 502, `{"error":"upstream_unavailable","source":"..."}`
- Rate limit: status 429, retry-after header

## 5. Tool-definitioner — komplette specs

Brug `zod` til input-skemaer. Hver tool eksporterer `{ name, description, inputSchema, handler }`.

### 5.1 `lookup_company`

```typescript
{
  name: "lookup_company",
  description: "Look up basic company data (name, address, status, industry, VAT registration, founding date) from official European business registries. Supports 14 countries: DK (CVR), NO (Brønnøysund), SE (Bolagsverket), FI (YTJ/PRH), NL (KVK), BE (KBO), IE (CRO), UK (Companies House), FR (INSEE Sirene), DE (Handelsregister), CZ (ARES), PL (KAS+KRS), LV (Uzņēmumu reģistrs), EE (Ariregister).",
  inputSchema: z.object({
    country: z.enum(["dk","no","se","fi","nl","be","ie","uk","fr","de","cz","pl","lv","ee"])
      .describe("ISO 3166-1 alpha-2 country code, lowercase"),
    id: z.string().min(1).describe("National company identifier. DK=CVR (8 digits), NO=orgnr (9), SE=orgnr (10), FI=Y-tunnus (NNNNNNN-D), NL=KVK (8), BE=10 digits, IE=CRO (1-7), UK=8 chars, FR=SIREN (9), DE=LEI (20) or HRB number, CZ=IČO (8), PL=NIP (10) or KRS (10), LV=11 digits, EE=8 digits.")
  })
}
```

Handler: `GET /api/company/{country}/{id}`

```typescript
{
```

```typescript
  description: "Look up a Legal Entity Identifier (LEI) via GLEIF — the global standard for entity identification. Returns legal name, registered address, status, parent + ultimate parent relationships, and child entities (subsidiaries). Also supports reverse lookup from a national company number to LEI.",
  inputSchema: z.object({
```

## 6. Code skeleton

### `src/lib/apiClient.ts`

```typescript
const BASE_URL = process.env.NORDIC_API_BASE_URL ?? "https://api.addonnordic.dk";
const API_KEY = process.env.NORDIC_API_KEY;

if (!API_KEY) {
  throw new Error("NORDIC_API_KEY environment variable is required. Get a key at https://addonnordic.dk");
}

export interface ApiError {
  status: number;
  error: string;
  source?: string;
  details?: unknown;
}

export async function apiGet<T = unknown>(path: string): Promise<T> {
  const url = `${BASE_URL}${path}`;
  const res = await fetch(url, {
    headers: { "X-API-Key": API_KEY!, "Accept": "application/json" },
  });
  if (!res.ok) {
    const body = await res.json().catch(() => ({}));
    throw { status: res.status, ...body } as ApiError;
  }
  return res.json() as Promise<T>;
}

export async function apiPost<T = unknown>(path: string, body: unknown): Promise<T> {
  const res = await fetch(`${BASE_URL}${path}`, {
    method: "POST",
    headers: {
      "X-API-Key": API_KEY!,
      "Accept": "application/json",
      "Content-Type": "application/json",
    },
    body: JSON.stringify(body),
  });
  if (!res.ok) {
    const errBody = await res.json().catch(() => ({}));
    throw { status: res.status, ...errBody } as ApiError;
  }
  return res.json() as Promise<T>;
}
```

### `src/index.ts` (stdio transport — for Claude Desktop, Cursor, Claude Code)

```typescript
#!/usr/bin/env node
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { CallToolRequestSchema, ListToolsRequestSchema } from "@modelcontextprotocol/sdk/types.js";
import { tools } from "./tools/index.js";

const server = new Server(
  { name: "nordic-data-mcp", version: "0.1.0" },
  { capabilities: { tools: {} } }
);

server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: tools.map(t => ({
    name: t.name,
    description: t.description,
    inputSchema: t.jsonSchema, // pre-converted from zod
  })),
}));

server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const tool = tools.find(t => t.name === request.params.name);
  if (!tool) throw new Error(`Unknown tool: ${request.params.name}`);
  try {
    const args = tool.inputSchema.parse(request.params.arguments);
    const result = await tool.handler(args);
    return { content: [{ type: "text", text: JSON.stringify(result, null, 2) }] };
  } catch (err: any) {
    return {
      content: [{ type: "text", text: `Error: ${err.message ?? JSON.stringify(err)}` }],
      isError: true,
    };
  }
});

const transport = new StdioServerTransport();
await server.connect(transport);
console.error("Nordic Data MCP server running on stdio");
```

### Tool eksempel: `src/tools/lookupCompany.ts`

```typescript
import { z } from "zod";
import { zodToJsonSchema } from "zod-to-json-schema";
import { apiGet } from "../lib/apiClient.js";

const inputSchema = z.object({
  country: z.enum(["dk","no","se","fi","nl","be","ie","uk","fr","de","cz","pl","lv","ee"]),
  id: z.string().min(1),
});

export const lookupCompany = {
  name: "lookup_company",
  description: "Look up basic company data (name, address, status, industry, VAT registration, founding date) from official European business registries. Supports 14 countries...",
  inputSchema,
  jsonSchema: zodToJsonSchema(inputSchema),
  handler: async (args: z.infer<typeof inputSchema>) => {
    return apiGet(`/api/company/${args.country}/${encodeURIComponent(args.id)}`);
  },
};
```

## 13. Hvad du IKKE skal gøre

- **Ikke** kopiere kode fra Nordic Data API backend. MCP-serveren skal kun lave HTTP-kald.
- **Ikke** caching i MCP-laget. Backend har allerede caching.
- **Ikke** business logic, scoring, eller transformations. Returner råt JSON fra backend.
- **Ikke** hardcode API-nøgle. Skal komme fra env.
- **Ikke** lave egen rate-limiting. Backend håndterer det.
- **Ikke** logge request bodies eller responses i prod (PII).
